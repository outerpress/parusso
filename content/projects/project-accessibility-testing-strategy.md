I developed a testing process to improve the team's knowledge of common accessibility issues and test all the changes we put in front of users.
{{< /intro >}}

### Background

Accessibility is a core design requirement for health tools whose users rely on assistive devices to navigate web tools at a higher rate than other populations. 

Our team had build accessibility criteria into automated tests, but it didn’t catch everything. And there were gaps in our teams knowledge of how to fix these issues.

### My work

I set up an accessibility testing checklist and trained UX and developers how to run through it. This checklist was useful at every stage of the process. But it was required acceptance criteria on every front-end ticket.

### Outcome

This led to developers testing front end work with screen readers along with other tools and fixing many issues that would have otherwise not been discovered.
